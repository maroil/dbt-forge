`dbt-forge` is a Python CLI that scaffolds new dbt projects, extends existing ones
with models, tests, and CI pipelines, validates project health, and keeps generated files
up to date as templates improve.

Use `init` to create a project. Use `add` to extend it. Use `doctor` to validate best
practices. Use `status` for a project overview. Use `update` to re-apply templates.
Use `--preset` to enforce team standards.

## How to use these docs

- Start with [Getting started](/docs/getting-started/) for the quickest path from install to a working dbt project.
- Use [dbt project template](/docs/guides/dbt-project-template/) when you want to evaluate what should go into a reusable dbt project scaffold.
- Use [How to scaffold a dbt project](/docs/guides/how-to-scaffold-a-dbt-project/) when you want a step-by-step implementation guide.
- Use [Migrate SQL to dbt](/docs/guides/migrate-sql-to-dbt/) when you are converting legacy SQL scripts into dbt.
- Use [dbt project best practices](/docs/guides/dbt-project-best-practices/) when you want a checklist for maintainable projects.
- Use [dbt Mesh project setup](/docs/guides/dbt-mesh-project-setup/) when you are planning a multi-project dbt Mesh scaffold.
- Use [init](/docs/cli/init/) when you are creating a new dbt project.
- Use [add](/docs/cli/add/) when you are extending an existing dbt project with new components.
- Use [doctor](/docs/cli/doctor/) when you want to validate project health and best practices.
- Use [status](/docs/cli/status/) when you want a quick overview of model counts, test coverage, and packages.
- Use [update](/docs/cli/update/) when you want to re-apply dbt-forge templates after upgrading.
- Use [preset](/docs/cli/preset/) when you want to enforce team standards across projects.
- Use [Project structure](/docs/project-structure/) when you need to understand the generated layout before editing it.
