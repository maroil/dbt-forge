`dbt-forge` is a Python CLI that scaffolds new dbt projects, extends existing ones
with models, tests, and CI pipelines, and validates project health with automated checks.

Use `init` to create a project. Use `add` to extend it with new components. Use `doctor`
to validate best practices.

## How to use these docs

- Start with [Getting started](/docs/getting-started/) for the quickest path from install to a working dbt project.
- Use [init](/docs/cli/init/) when you are creating a new dbt project.
- Use [add](/docs/cli/add/) when you are extending an existing dbt project with new components.
- Use [doctor](/docs/cli/doctor/) when you want to validate project health and best practices.
- Use [Project structure](/docs/project-structure/) when you need to understand the generated layout before editing it.
